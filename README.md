# 羽毛球 Drop-in 账号分配小程序

这是一个微信原生小程序 + 云开发项目，用于在羽毛球馆 drop-in 场景中统一管理当天有效的临时用户名/密码，并记录账号当前在哪个场地排队或打球。

## 当前版本功能

- 微信云开发登录识别用户
- 通过分享到已有微信群来创建/加入对应羽毛球群
- 单页添加当天用户名/密码
- 账号池动态排序：空闲、正在打、排队中
- 选择 2 个或 4 个空闲账号加入场地队列
- 每组默认 45 分钟
- 群成员可取消任意排队/正在打记录
- 自动释放到时账号，并推进下一组

## 微信开发者工具设置

1. 打开微信开发者工具，选择“导入项目”。
2. 项目目录选择本文件夹：`O:\WeChatProjs\PasscodeManager_WeChat`。
3. AppID 填你自己的小程序 AppID。
4. 开通“云开发”，创建一个云环境。
5. 打开 `miniprogram/app.js`，把 `cloudEnvId` 改成你的云环境 ID。注意它不是小程序 AppID；如果只有一个云环境，也可以先留空。
6. 在微信开发者工具里右键 `cloudfunctions/api`，选择“上传并部署：云端安装依赖”。
7. 在云开发控制台创建集合：
   - `users`
   - `groups`
   - `credentials`
   - `courts`
   - `queueEntries`
   - `operationLogs`
8. 数据库权限建议先设为“仅创建者可读写”或更严格；业务读写主要走云函数。
9. 部署后编译运行小程序。

## 绑定微信群

微信小程序不能直接弹出群列表让用户选择。当前流程是：

1. 在首页点击“选择微信群”。
2. 在微信转发面板里选择已有羽毛球微信群。
3. 从微信群里的小程序卡片打开。
4. 小程序会自动用该微信群的 `openGId` 创建或加入对应的排队群。

注意：微信开发者工具里的“查看分享页”通常不会模拟真实微信群的 `openGId`。这个流程需要用真机预览或体验版测试，把卡片发到微信群后从群聊里打开。

实现细节：`wx.cloud.CloudID` 必须作为云函数调用的顶层字段传入，不能嵌套在 `payload` 里面，否则云函数端拿不到解密后的 `openGId`。

## 定时自动推进

云函数 `api` 支持 `advanceExpired` 动作。上线后可在云开发控制台给该云函数增加定时触发器，每分钟执行一次，事件内容：

```json
{
  "action": "advanceExpired",
  "payload": {}
}
```

开发阶段即使不配定时器，每次打开页面或刷新数据也会触发一次自动推进。
